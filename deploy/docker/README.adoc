= Host Docker Infrastructure

This Docker layer is host-owned infrastructure for the local `App` runtime.
It is intentionally shared across connected Symfony bundles and must not be duplicated per component.

== Scope

The current baseline provides:

* PostgreSQL for business and user data
* Redis for cache / transient state / queue-adjacent runtime concerns
* RabbitMQ for asynchronous messaging transports
* Elasticsearch for search and projection workloads
* Mailpit for local email capture
* Mercure for real-time updates

SQLite remains a local host file and is not containerized.

== Location

Docker assets live under `deploy/docker/` so deployment/runtime wrappers remain separate from the application root.

== Start

[source,bash]
----
docker compose --env-file deploy/docker/.env -f deploy/docker/compose.yaml up -d
----

Or on Windows:

[source,powershell]
----
powershell -ExecutionPolicy Bypass -File .\deploy\docker\up.ps1
----

== Stop

[source,bash]
----
docker compose --env-file deploy/docker/.env -f deploy/docker/compose.yaml down
----

== Stop and remove volumes

[source,bash]
----
docker compose --env-file deploy/docker/.env -f deploy/docker/compose.yaml down -v
----

== Logs

[source,powershell]
----
powershell -ExecutionPolicy Bypass -File .\deploy\docker\logs.ps1
----

== Default exposed ports

* PostgreSQL: `127.0.0.1:5432`
* Redis: `127.0.0.1:6379`
* RabbitMQ AMQP: `127.0.0.1:5672`
* RabbitMQ Management UI: `127.0.0.1:15672`
* Elasticsearch HTTP: `127.0.0.1:9200`
* Mailpit SMTP: `127.0.0.1:1025`
* Mailpit UI: `127.0.0.1:8025`
* Mercure HTTP: `127.0.0.1:3000`

== Host alignment

The host `.env` should point to these local endpoints, for example:

[source,env]
----
DATABASE_URL="postgresql://app:!ChangeMe!@127.0.0.1:5432/app?serverVersion=16&charset=utf8"
APP_DATA_DATABASE_URL="sqlite:///%kernel.project_dir%/var/data/app_data.sqlite"
REDIS_URL="redis://127.0.0.1:6379"
MESSENGER_TRANSPORT_DSN="amqp://guest:guest@127.0.0.1:5672/%2f/messages"
ELASTICSEARCH_URL="http://127.0.0.1:9200"
MAILER_DSN="smtp://127.0.0.1:1025"
MERCURE_URL="http://127.0.0.1:3000/.well-known/mercure"
MERCURE_PUBLIC_URL="http://127.0.0.1:3000/.well-known/mercure"
----

== Notes

* Infrastructure is started once for the host app, not once per component.
* Components consume host-defined DSNs and service endpoints.
* If Messaging uses RabbitMQ, Redis, Mercure and Elasticsearch, all of them are still host-level services.
